CMResponse Class Reference

Inherits from NSObject
Declared in CMResponse.h

Overview

CMResponse contains a pointer back to the CMRequest that generated it, as well as numerous conveniences that allow quick introspection of the HTTP response and any transformed results or NSError objects.

CMResponse.h

expectedContentLength

The expected length of the response body reported by the server.

@property (nonatomic, readonly) long long expectedContentLength

Return Value

The content length or NSURLResponseUnknownLength.

Discussion

If there is a ‘Content-Length’ response header, its value is used to represent this attribute. Otherwise, if there was a ‘Content-Range’ header, the difference between the start and end bytes is returned. If it is not possible to calculate contentLength by either method (such as when the response reflects streamed content), NSURLResponseUnknownLength is returned.

Declared In

CMResponse.h

expectedContentRange

The expected range of the selected content to be sent back by the server in response to a range request.

@property (nonatomic, readonly) CMContentRange expectedContentRange

Discussion

When the server sends back part of the selected content it includes a ‘Content-Range’ header with a value similar to: ‘bytes 0-10000/50000, which corresponds the start and end of the range over the total bytes of the content. This property returns each of the three distinct data points in the location,length and contentLength of a CMContentRange.

%note%
Note: If there was no ‘Content-Range’ header, the returned range will have a location of kCFNotFound.
